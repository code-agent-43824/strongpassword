О генераторе паролей

Как работает генератор

Вы выбираете длину и наборы символов. Браузер создаёт случайные байты через Web Crypto API и собирает пароль на вашем устройстве.

Почему пароль генерируется локально

Локальная генерация убирает лишний риск: пароль не уходит по сети, не попадает в серверные журналы и не обрабатывается backend-кодом.

Какая длина пароля считается безопасной

Для обычных аккаунтов разумный минимум — 16 символов. Для финансов, админок и ключевых сервисов лучше использовать 20–32 символа и менеджер паролей.

Зачем исключать похожие символы

Опция убирает символы вроде O, 0, I и l. Это снижает риск ошибки, если пароль нужно продиктовать, перепечатать или сохранить вручную.

Для чего использовать разные типы паролей

У разных сервисов разные ограничения. Профили помогают быстро выбрать длину и символы для обычного аккаунта, банка, сервера или резервного кода.

Почему не стоит отправлять пароль на сервер

Серверная генерация создаёт дополнительную точку доверия. Для пароля безопаснее, когда секрет появляется только в браузере и сразу остаётся у вас.
